# README Translation Policy

GeoLoop treats translation as product documentation, not as decorative text.

## Policy

1. `README.md` is the source of truth.
2. Localized README files should preserve product logic, safety rules and roadmap links.
3. Translations may adapt wording for local readability, but must not change claims.
4. No language version may add unsupported local market proof.
5. Safety boundaries must remain equally strict in every language.
6. AI-generated translations require human review before major public promotion.

## Claim Control

Forbidden in localized README files:

- fake local customers
- fake local rankings
- fake App Store results
- fake AI search citations
- fake benchmark numbers
- unsupported market leadership claims

## Review Cadence

Review localized README files when:

- product positioning changes
- roadmap changes
- safety policy changes
- CLI commands change
- GitHub Action behavior changes
- pricing or hosted plan strategy changes

## Drift Handling

If a localized README is stale, either update it immediately or add a visible note:

```text
This translation may lag behind README.md. Please check the English README for the latest source of truth.
```
