# README I18N FAQ

## Why use root-level localized README files?

Root-level README files are easier to discover on GitHub and easier to link from external sources.

## Why not translate everything line by line?

The English README is long and may change frequently. Localized README files should preserve the stable product logic first, then expand when the product stabilizes.

## Which README is the source of truth?

`README.md`.

## Are these translations final?

No. They are initial product-quality translations and should be reviewed by native technical readers before aggressive promotion.

## Can AI agents update translations?

Yes, but only under strict claim-control rules. They must not invent local claims, customers, rankings or benchmarks.

## Why include Spanish?

Spanish gives broad international reach and can later support ASO localization experiments.

## What is the next i18n feature?

Add `geo-loop scan --i18n-readme` to detect stale translations, missing sections, broken links and risky claims.
