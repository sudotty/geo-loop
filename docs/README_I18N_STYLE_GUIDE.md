# README I18N Style Guide

## Tone

Use clear, technical, founder/developer-friendly language.

Avoid hype.

## Technical Terms

Keep these terms recognizable:

- GitHub-native
- SEO
- GEO
- ASO
- GitHub Actions
- Copilot
- AI Agent
- Pull Request
- Issue
- App Store

Translate surrounding explanation, but preserve terms when translation would reduce clarity.

## Sentence Style

Prefer:

- short definitions
- direct claims
- tables
- loops and diagrams
- explicit safety boundaries

Avoid:

- vague marketing copy
- exaggerated promises
- local market claims without evidence
- AI-sounding filler

## Safety Wording

Every localized README should clearly state:

- no guaranteed AI ranking
- no fake claims
- no automatic App Store submission by default
- human review for risky changes
