# ASO Rationale for Multilingual README Files

GeoLoop's multilingual README files also support future ASO workflows.

## Why ASO Needs GitHub Docs

For app products, ASO metadata should not live in isolation. App Store title, subtitle, keyword ideas, screenshot copy and release notes should connect to:

- product positioning
- website copy
- README language
- docs language
- competitor comparison pages
- user problem language

A multilingual README creates a stable semantic base for ASO experiments.

## Supported ASO Outputs

Future GeoLoop ASO modules can generate:

- app name candidates
- subtitle candidates
- keyword field ideas
- screenshot copy
- release note drafts
- localization drafts
- competitor metadata summaries
- review-language summaries

## Language Strategy

Start with languages that match likely product distribution:

- English
- Simplified Chinese
- Japanese
- Korean
- Spanish

Add more languages only when there is a real market or distribution reason.

## Safety

GeoLoop should never automatically submit App Store changes by default. It should generate metadata drafts for human review.
