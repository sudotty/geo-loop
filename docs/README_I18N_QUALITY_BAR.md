# README I18N Quality Bar

A localized README is acceptable when it meets these standards.

## Minimum Quality

- product definition is correct
- SEO / GEO / ASO are explained correctly
- GitHub-native loop is preserved
- current status is not overstated
- safety boundary is present
- links work
- no fake local claims

## Good Quality

- language is natural for developers
- examples are locally understandable
- technical terms are preserved where useful
- roadmap links are clear
- target user table is readable

## Excellent Quality

- native technical review completed
- localized examples added without unsupported claims
- local SEO/GEO prompt set created
- ASO localization draft connected
- performance measured by market

## Current Level

The current localized README files meet the minimum quality bar and are suitable as initial public documentation.
