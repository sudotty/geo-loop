# README I18N Maintainers Guide

## Maintainer Responsibilities

When changing `README.md`, maintainers should check whether localized README files need updates.

## Update Triggers

Update localized README files when changing:

- product definition
- CLI commands
- installation flow
- current status
- roadmap
- safety policy
- supported channels
- hosted product strategy
- pricing or plan language

## Review Checklist

- [ ] localized README files still match product reality
- [ ] language navigation is correct
- [ ] no broken links
- [ ] no fake local claims
- [ ] no stale CLI commands
- [ ] safety boundaries remain strict

## Recommended Workflow

```text
README.md change -> i18n check -> localized README PR -> human review -> merge
```
