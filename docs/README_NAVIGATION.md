# README Navigation

Use these root-level files for language-specific entry points:

| Language | File |
|---|---|
| English | [`README.md`](../README.md) |
| Simplified Chinese | [`README.zh-CN.md`](../README.zh-CN.md) |
| Japanese | [`README.ja.md`](../README.ja.md) |
| Korean | [`README.ko.md`](../README.ko.md) |
| Spanish | [`README.es.md`](../README.es.md) |

## Recommended Main README Block

When editing the main `README.md`, keep a language block near the top:

```md
## Languages

- [English](README.md)
- [简体中文](README.zh-CN.md)
- [日本語](README.ja.md)
- [한국어](README.ko.md)
- [Español](README.es.md)
```

## Why Not Put Translations Under `docs/`?

Root-level localized README files are easier to discover from GitHub, easier to link, and more likely to be treated as first-class project entry points.
