# I18N README Checklist

Use this checklist when updating localized README files.

## Required Checks

- [ ] Language navigation exists and links are correct
- [ ] Product definition matches `README.md`
- [ ] SEO / GEO / ASO meanings are consistent
- [ ] Core loop is preserved
- [ ] Target users table is preserved
- [ ] Current status is accurate
- [ ] Roadmap links are correct
- [ ] Safety boundary is not weakened
- [ ] No fake local claims were introduced
- [ ] No unsupported local market claims were introduced

## Agent Prompt

```text
Update the localized README file to match the product logic in README.md.
Preserve factual accuracy.
Do not add fake claims, customers, benchmarks, rankings or local market proof.
Keep the tone clear, technical and founder/developer friendly.
Keep SEO, GEO and ASO technical terms recognizable.
If a section cannot be confidently localized, preserve the English technical phrase.
```

## Recommended Review

A human reviewer should check:

- terminology
- local readability
- factual consistency
- links
- safety language
- no unintended claims
